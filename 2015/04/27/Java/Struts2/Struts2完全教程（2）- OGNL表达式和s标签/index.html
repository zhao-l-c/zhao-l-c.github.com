Struts2的OGNL

OGNL是Object Graphic Navigation Language（对象图导航语言）的缩写，它是一个开源项目。 Struts 2框架使用OGNL作为默认的表达式语言。

下面介绍Struts2使用的OGNL：

Context

OGNL 有一个上下文（Context）概念，说白了上下文就是一个MAP结构，它实现了java.utils.Map接口。在Struts2中上下文的实现为ActionContext，下面是上下文的结构示意图：

上下文中除了application、session、request、parameters和attr外，还包括了value stack和current action这两个重要的对象。

值栈

OGNL还有一个ValueStack，称为值栈。值栈在Struts2中的实现类为 OgnlValueStack。值栈中有两个逻辑部分，分别是context和root，其中：

  • context是上下文对象，下面称它为Map栈，它的类型为OgnlContext，其实它的本质是就是ActionContext的一个引用；
  • root称为值栈的根对象，下面称它为对象栈，它的类型为CompoundRoot，它是继承了ArrayList实现的一个栈结构。 对象栈中的栈顶是当前Action类对象（可以通过标签查看）。

OgnlValueStack和ActionContext的关系：
从前面的分析可知，Struts源码中，OgnlValueStack可以通过ActionContext获取，同时 ActionContext也是 OgnlValueStack 的一个成员。

区分值栈、对象栈和Map栈的概念，其中值栈包含了对象栈和Map栈。

如何获取值栈中的属性值

Struts2是利用s:property标签和OGNL表达式获取值栈中的属性值。

注意，由于值栈有2部分组成，因此获取属性值分为：

  • 对于对象栈，是获取栈中某个对象的属性。
  • 对于Map栈，是获取application、session等对象中的属性值。

从对象栈获取属性

从Map栈获取属性

此时需要使用#符号标注命名空间。其中：

  • application： 用于访问 ServletContext对象，例如<s:property value=" #application.userName "/>相当于调用ServletContext. getAttribute(" userName ")， 对应的EL表达式写法为\${applicationScope.userName}；
  • session：用于访问HttpSession对象，例如<s:property value=" #session.userName "/>相当于调用HttpSession .getAttribute("userName") ， 对应的EL表达式写法为\${sessionScope.userName} ；
  • request： 用于访问 HttpServletRequest对象，例如<s:property value=" #request.userName "/>相当于调用HttpServletRequest .getAttribute("userName") ， 对应的EL表达式写法为\${requestScope.userName} ；
  • parameters：用于访问HTTP的请求参数，例如<s:property value=" #parameters.userName "/>相当于调用request.getParameter("username") ， 对应的EL表达式写法为\${param.userName} ；
  • attr：用于按page->request->session->application顺序访问属性。

如果Map中设置的属性是对象，还可以一级一级的往下获取，例如：<s:property value="#application.product.productName"/>。

为什么可以使用EL表达式获取属性值？

Struts2 的项目中，JSP页面最后同样是解析为Servlet代码，但是Struts2 会把请求对象封装为StrutsRequestWrapper类型，查看它的继承关系可以发现它是HttpServletRequest的实现类。因此在执行解析后的Servlet中的 _jspService方法时，会把这个封装类作为参数传递进去。

主题

Struts2 的标签是有主题的，也就是Struts2会根据标签的主题格式化生成的HTML代码。Struts2有以下几种主题：

  • simple：把 UI 标签翻译成最简单的 HTML 对应元素，而且会忽视行标属性；
  • xhtml：是默认的主题，这个主题的模板通过使用一个布局表格提供了一种自动化的排版机制；
  • css_xhtml：这个主题里的模板与 xhtml 主题里的模板很相似, 但它们将使用 css 来进行布局和排版；
  • ajax：这个主题里的模板以 xhtml 主题里德模板为基础, 但增加了一些 Ajax 功能。
