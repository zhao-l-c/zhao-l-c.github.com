安装和使用 ###安装maven

把下载文件解压缩，并把bin文件夹所在目录添加在环境变量中就安装完毕。Windows下打开命令行窗口，输入mvn -version查询版本信息。

修改maven 的本地仓库

默认安装后的仓库地址是${user.home}/.m2/repository，可以对它进行修改，方法是：
进入maven的conf目录下的setting.xml文件，添加：
<localRepository>D:/maven/maven_repository</localRepository>
（并把该配置文件setting.xml 复制到新的仓库地址的位置的仓库文件夹的上一级文件，例如这里要复制到 D:/maven/ 下—这一步应该不需要—）

HelloWorld Maven

可以使用IDE帮助创建Maven项目，直接创建，只要满足Maven的目录结构就行。

maven的目录结构

maven构建Java项目时，要求Java代码保存在规定的目录结构下，也就是：

  • 源文件所在目录为：[root]/src/main/java
    源文件使用的资源文件放到：[root]/src/main/resource

  • 测试代码放在：[root]/src/test/java
    测试代码的资源文件放到：[root]/src/test/resource

  • 执行命令后生成的文件放在[root]/target，这些文件包括class文件，测试报告文件，jar包，war包等。

最后，在Maven项目的根目录下应该提供pom.xml文件！下面介绍这个文件的内容。

隐式变量

Maven提供了三个隐式的变量可以用来访问环境变量，pom信息，和Maven Settings。

  • env
    env变量，暴露了你操作系统或者shell的环境变量。便 如在Maven POM中一个对\${env.PATH}的引用将会被\${PATH}环境变量替换，在Windows中为%PATH%。
