上面的代码中在execute方法中创建Thread匿名内部类的方式新建线程，因此每一个线程都是一个线程对象，所以在execute方法中的局部变量都是独立的，并不会有线程安全的问题，例如示例代码中的tmpData变量。
而通过方法的参数传递进来的变量，它们都是final修饰的，并对于基本类型变量则不能重新赋值，而对于引用类型变量，除了直接引用的变量不能重新赋值外，变量中的属性是可以改变的。因此引用类型参数对象其实是线程共享的。所以示例中的Data参数对象调用getNumber和handle方法时，是在一个同步代码块中调用，否则有线程安全问题。

抛出InterruptedException的方法

Java标准API中的方法会抛出InterruptedException异常的代表方法有：

java.lang.Object类的wait
java.lang.Thread类的sleep
java.lang.Thread类的join

这3个方法的特定是都需“要花费时间”，例如执行wait方法的线程，在等待期间不执行；执行jon的线程，会等待指定的线程执行完毕才继续执行。
当然它们也可以取消等待，那就是执行该线程的interrupt方法。

2. wait与interrrupt

同样，对执行了wait的方法的线程实例执行interrupt方法，会给线程传达“不用等notify/notifyAll了，从等待区出来把”的意图。
当对wait中的线程执行interrupt方法时，会先重新获取锁定，然后抛出InterruptedException异常。

注意：执行wait方法的线程必须获取锁，否则会抛出IllegalMonitorStateException异常。

notify与interrupt的区别
这两个方法意义有点接近，但还是有差别的：
1）notify/notifyAll是Object类的方法，调用该方法需要获取锁定。而interrupt是Thread的方法，直接对线程对象调用。
2）被notify/notifyAll唤醒的线程执行wait的下一句代码，而被interrupt唤醒的线程则转向catch中继续执行。

3. join与interrupt

类似sleep方法，不需要获取锁。
