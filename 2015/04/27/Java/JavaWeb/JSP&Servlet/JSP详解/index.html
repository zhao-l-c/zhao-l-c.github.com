什么是JSP

一句话概括 JSP：它是 Java 程序和 Html 标签的集合体，最后会被翻译成一个 Servlet 程序。

当WEB 容器接收到以 .jsp 为扩展名的 URL 的访问请求时，它将把该访问请求交给 JSP 引擎去处理。每个 JSP 页面在第一次被访问时，JSP 引擎将它翻译成一个 Servlet 源程序，再把它编译成class 类文件，然后由 WEB 容器像调用普通 Servlet 程序一样

JSP的隐含对象

JSP包括了九大隐含对象：

隐含对象 类型
application ServletContext
session HttpSession
request HttpServletRequest
response HttpServletResponse
pageContext PageContext
page 当前JSP翻译后的Servlet，也就是this
out JspWriter
config ServletConfig
exception Throwable

注意：

  • 所谓 隐含对象，也就是在JSP页面中不用定义就可以使用的变量对象；
  • 隐含对象是翻译后的源程序的_jspService方法中创建 ，可以发现request和response是该函数的参数，而剩余的7个对象则作为函数的局部变量并初始化；
  • exception隐含对象只有在JSP指令<%@ page %>中声明属性isErrorPage="true"后才会出现；
  • pageContext，request，session，application称为域对象，这4个域对象都有setAttribute和getAttribute方法，它们的作用域分别 从小到大。

下面分别介绍这些隐含对象：

pageContext

PageContext类型，它代表当前页面的上下文，在pageContext设置的属性作用域是当前页面，不能转发或者重定向到别的页面。

request

HttpServletRequest类型，它代表一个请求，在request中设置的属性可以转发到别的Servlet或者这JSP。例如：

session

HttpSession类型，它代表一个会话， 同一次会话的多个资源都可以共享（前提是不要关闭浏览器）。也就是在session中设置的属性可以被多个会话共享。

application

ServletContext类型，代表当前应用的上下文， 在不重启服务器的前提下，设置到application中的属性可以在整个应用中共享。

out

JspWriter类型，可以直接输出内容到JSP页面。

了解知识： 区分 out与 response.getWriter()
response.getWriter()也提供了输出内容到JSP页面的方法。两者的区别是：

  • out是javax.servlet.jsp. JspWriter类型，而另一个是java.io.PrintWriter类型；
  • response.getWriter() 仅支持输出字符类型数据，即字符、字符数组、字符串等；而out 可以将各种类型的数据通过默认编码转换成bytes字节形式，最后通过write(int c)方法被输出；
  • out是带缓冲区的，可以使用out.getBufferSize()获取缓冲区的大小，也就是out不是 直接将数据输出到页面，而是将数据刷新到response的缓冲区后再输出 ；而 response.getWriter () 不带缓冲区， 直接输出数据。

<%@ page %>

对 pageEncoding 和 contentType补充说明：

  • pageEncoding：声明把 JSP 文件翻译为 Java 源码后的编码格式。 而如果 pageEncoding 的值和 JSP 保存的编码格式（Eclipse中可以通过右键properties 查看）不一致，那么翻译后的 Java 文件就会出现乱码。

  • contentType：声明翻译后的源码发送到浏览器时的数据类型和编码格式。 所以 pageEncoding 和 contentType 即使不同也没有问题，关键是 pageEncoding 要和 JSP 文件的保存编码类型一定一致。
