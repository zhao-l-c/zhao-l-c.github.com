    • 网络协议

    UDP

    特点：
    1）不需要建立链接；
    2）将数据及源和目的封装在包中，大小限制在64K ；
    3）不可靠，速度快 ；
    4）应用于视频会议、聊天、桌面共享等。

    TCP

    特点：
    1）建立连接，形参传输数据的通道 ；
    2）通过3次握手完成连接，是可靠的 ；
    3）效率较低，但可以大数据量传输，不需要封装包 ；

    UDP编程

    1. API简介

    DatagramSocket：用于发送或接收的socket服务，注意接收端需要指定接收的端口号，而发送端可以不指定端口号，由系统自动分配。
    DatagramPacket ：用于创建发送或接收服务的数据包，注意用于封装发送端的数据包时，需要指定目的地址的端口号，
    这个端口号与上面的DatagramSocket是不一样的意义。

    2. 使用说明

    UDP分发送端和接收端，它们都对应DatagramSocket，传输的时候都使用DatagramPacket。
    发送端：
    1）构建DatagramSocket服务，可以不指定端口号
    2）封装DatagramPacket 数据包，需要指定端口号
    3）发送数据
    4）关闭资源

    接收端：
    1）构建DatagramSocket服务，需要指定监听的端口号
    2）封装DatagramPacket 数据包，不需要端口号
    3）接收数据，接收方法是阻塞等待数据的
    4）关闭资源

    TCP编程

    使用说明

    TCP分客户端和服务端，客户端使用的是Socket，服务端使用的是ServerSocket。

    客户端：
    1）创建Socket服务对象，指定目的主机和端口号。创建的过程中就要进行3次握手过程。
    2）若连接建立后就能获取socket数据流，从socket数据流中就能获取输入和输出流。
    注意：
    1）客户端对象创建Socket对象时一般需要指定主机和端口号，同时系统也会为这个客户端程序分配端口号。
    2）只有服务端已经启动客户端才有可能创建成功Socket对象。
    3）从socket的输入流中读取数据时也是阻塞的。

    服务端：
    1）创建ServerSocket对象，指定监听端口号。
    2）通过accept方法获取连接过来的客户端Socket对象，若没有则等待，说明该方法是阻塞的。
    3）对客户端对象进行操作，完成后可以关闭客户端对象。

    注意：客户端与服务端通信时，客户端通过输出对象out输出数据，服务端此时要获取客户端对象的in对象获取数据；
    服务端利用客户端的out对象向客户端输出数据，客户端就要从in对象中获取这些数据。
