概述

命令（Command）模式：又称Action模式或者Transaction模式。命令模式把发出命令的负责人和执行命令的负责人分开，也就是实现功能的分工。
命令模式包含下面几个角色：
Client：创建命令并决定接收者。
Command：命令接口，定义抽象方法。
ConcreteCommand：具体命令，负责调用接收者的相应操作。
Invoker：请求者，负责调用命令对象执行请求。
Receiver：接收者，负责具体实施和执行一次请求。
