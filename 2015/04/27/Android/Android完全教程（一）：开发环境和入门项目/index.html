搭建开发环境

Eclipse+ADT

  • 首先安装JDK和Eclipse；

  • 安装Android SDK：
    Android SDK包含了开发Android应用所依赖的jar文件、运行环境及相关工具。下载文件：http://dl.google.com/android/android-sdk_xxx-windows.zip 并解压到任意目录，最后把android-sdk-windows\tools添加到系统环境变量中即可；

  • 在Eclipse中安装ADT：
    ADT（Android Development Tools）是为Eclipse开发Android应用定制的插件，下载文件：http://dl.google.com/android/ADT-xxx.zip。
    安装步骤：
    启动Eclipse，选择 Help-> Install New Software，点击Add按钮，在对话框的name一栏输入”ADT”， 然后点击”Archive…”，选择已经下载的ADT插件压缩文件，点击OK，点击 Next，最后重启 Eclipse。
    重启Eclipse后，选择window->preferences，在打开的视图左边点击android，在右边的SDK Location中选择Android SDK解压目录。

IntelliJ IDEA

使用IntelliJ IDEA则只需要安装Android的SDK即可，因为IDEA中有Android的插件。

ADT Bundle

还有一种最简单的方法，直接下载使用ADT Bundle。ADT Bundle包含了Eclipse、ADT插件和SDK Tools，是已经集成好的IDE，只需安装好Jdk即可开始开发。

AndroidDevTools

常用工具DBMS和ADB的使用

DBMS

DBMS（dalvik debug manager system），这是一个模拟器的调试器，所以不可以调试真机。

DBMS下常用的工具：

  • Devices工具
    列出了当前模拟器中的所有应用，包括系统和用于自己安装的应用，在Devices工具栏可以对这些应用进行操作。而且根据应用名可以使用adb uninstall [完整应用名]命令卸载该应用；
  • Emulator Control
    模拟器，包含对手机信号、网络连接、短信功能、GPS功能的模拟。
    其中若希望模拟器能上网，则需要配置主机电脑的IP和掩码。
  • LogCat
    日志输出。
  • File Explore
    这是模拟器的文件目录，一般包括3个文件：
    • data：模拟器的数据文件目录，该目录下的data就是存放安装的应用；
    • mnt：模拟器的SD卡，可以把里面的文件复制到本地电脑上或者把电脑上的文件复制到这里；
    • system：模拟器系统目录；

adb简单使用

首先需要把adb.exe所在的目录（例如D:\program\android-sdk\platform-tools）加到环境变量中，以便在CMD中直接使用！

常用命令：

  • 查看当前的设备，包括真机和模拟器：
    adb devices：

  • 关闭与开启服务，在出现连接错误时使用：
    adb kill-server
    adb start-server

  • 安装与卸载应用.apk文件：
    adb install [当前目录下的APK包名]
    adb uninstall [应用完整包名]
    注意：完整包名可以在DBMS工具的Devices 中查看。

  • 一般查询：
    adb version：查看adb 版本；
    adb devices：查看当前所有连接上来的设备信息

  • adb shell命令：
    ls：查看目录
    ps：查看进程运行信息
    kill [pid]：杀死进程

  • 查看adb端口是否被其它程序占用：
    netstat -aon|findstr "5037"：
    TCP 127.0.0.1:5037 0.0.0.0:0 LISTENING 12345
    查看监听端口5037的程序。
    接着tasklist|findstr "12345"
    tadb.exe 12345 Console 1 5,864 K

Android项目的基本结构

使用IDE工具创建Android项目，新创建的Android应用包含以下文件/目录：

  • src 目录
    Java源代码存放目录。
  • gen 目录
    存放所有由Android开发工具自动生成的文件。目录中最重要的就是R.java文件。 这个文件由Android开发工具aapt自动产生的。
    R.java在应用中起到了字典的作用，它包含了各种资源的id，通过R.java，应用可以很方便地找到对应资源。另外编绎器也会检查R.java列表中的资源是否被使用到，没有被使用到的资源不会编绎进软件中，这样可以减少应用在手机占用的空间。
    注意，在IntelliJ IDE中R文件的内容由IDE生成，实际文件是在类似~\.IntelliJIdea14\system\compiler\YOUR_PROJECT_NAME.RANDOM_STRING\.generated\aapt\YOUR_PROJECT_NAME.RANDOM_STRING\production\package\R.java这样的目录下。
  • res 目录
    资源文件目录， 在这个目录中我们可以存放应用使用到的各种资源，如xml界面文件，图片或数据。
  • res/drawable
    新建Android工程，工程一般会自动生成一个drawable的目录（若没有则手动生成）。 到了Android4.0时代，工程会自动生成drawable-hdpi, drawable-ldpi, drawable-mdpi, drawable-xhdpi四个目录。这4个目录对应不同dip大小时候选择的图片资源，即：

  • assets资源目录
    除了res目录存放资源文件外，在assets目录也可以存放资源文件，而且assets目录下的资源文件不会在R.java自动生成ID，所以读取assets目录下的文件必须指定文件的路径，如： file:///android_asset/xxx.3gp。

    项目清单文件， 这个文件列出了应用程序所提供的功能。如果应用使用到了系统内置的应用(如电话服务、互联网服务、短信服务、GPS服务等等)，你还需在该文件中声明使用权限。

Android中的显示单位：

常用的单位为：

  • px（pixels）像素，同一个像素值在不同像素的手机上显示不一样；

  • dip或dp（device independent pixels）设备独立像素，和不依赖手机具体像素，推荐使用这个单位。

  • sp（scaled pixels）比例像素，主要处理字体的大小，可以根据系统的字体自适应。

补充内容： fill_parent和wrap_content的区别：

  • fill_parent
    API Level 8及后续版本使用match_parent代替。
    设置一个构件的布局为fill_parent将强制性地使构件扩展，以填充布局单元内尽可能多的空间。设置一个顶部布局或控件为fill_parent将强制性让它布满整个屏幕。

  • wrap_content
    设置一个视图的尺寸为wrap_content将强制性地使视图扩展以显示全部内容。以TextView和ImageView控件为例，设置为wrap_content将完整显示其内部的文本和图像。布局元素将根据内容更改大小。
