数据库连接池

数据库连接池的基本思想就是为数据库连接建立一个“缓冲池”。预先在缓冲池中放入一定数量的连接，当需要建立数据库连接时，只需从“缓冲池”中取出一个，使用完毕之后再放回去。

JDBC的数据库连接池使用javax.sql.DataSource来表示，DataSource只是一个接口，该接口通常由服务器（Weblogic, WebSphere, Tomcat）提供实现，也有一些开源组织提供实现，如DBCP和C3P0数据库连接池。

使用数据源时，Connection执行close方法并没有关闭连接，而是把它归还到连接池中！

dbutils

commons-dbutils是Apache组织提供的一个开源 JDBC工具类库，它是对JDBC的简单封装，并且使用dbutils能极大简化JDBC编码的工作量。

org.apache.commons.dbutils.ResultSetHandler接口

如果要查询记录就需要用到QueryRunner的query方法，这个方法需要一个ResultSetHandler接口的实现类作为参数，ResultSetHandler接口表示对查询结果集的处理方式，例如返回一个bean实体类对象、返回List集合对象或者返回列名和列值的Map集合等等。

可以自定义ResultSetHandler实现类，同时dbutils也提供了很多实现类。

ResultSetHandler使用到了面向接口的设计模式。
